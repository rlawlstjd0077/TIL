## 소프트웨어 설계

### XP(eXtreme Programming)

개요

- 수시로 발생하는 고객의 요구사항에 유연하게 대응하기 위해서 고객의 참여와 개발 과정의 반복을 극대화하여 개발 생산성을 향상시키는 기법
- 5가지 핵심 가지: 의사소통, 단순성, 용기, 존중, 파드백

주요 실천 방법

- Pair Programming
- Test-Driven Development
- Whole Team
- Continuous Integration
- Design Improvement
- Small Release
- Collective Ownership

With .. Scrum

### 럼바우 분석 기법

개요

- 소프트웨어 구성 요소를 그래픽 표기법으로 이용하여 모델링하는 것
  - 객체 모델링 기법이라고도 함

종류

- 객체 모델링
  - 객체 다이어그램 (객체 관계)
- 동적 모델링
  - 상태 다이어그램을 이용해 시간에 흐름에 따른 객체들 간의 동적인 행위 모델링
- 기능 모델링
  - 자료 흐름도(DFD)를 이용해서 다수의 프로세스들 간의 자료 흐름을 중심으로 처리

### 아키텍쳐 패턴

개요

- 아키텍쳐를 설계할 때 참조할 수 있는 전형적인 해결방식, 예제

종류

- 클라이언트/서버 패턴 - 하나의 서버, 다수의 클라이언트 컴포넌트로 구성되는 패턴
- 파이프-필터 패턴 - 데이터 스트림 절차의 각 단계를 필터 컴포넌트로 캡슐화하여 파이프를 통해서 데이터를 전송하는 패턴
  - ex) UNIX의 쉘
- MVC 패턴

### 코드

개요

- 컴퓨터를 이용하여 자료를 처리하는 과정에서 분류, 조합 및 집계를 용이하게 하고 특정 자료의 추출을 쉽게 하기 위해서 사용하는 기호

기능

- 식별 기능, 분류 기능, 배열 기능

종류

- 순차 코드: 일정 기준에 따라 최초부터 순차적으로 일련 번호를 부여하는 방법
- 블록 고드: 대상 항목에서 공통적인 것을 블록으로 구분하고 블록 내에서 일련 번호를 부여하는 방법
- 10진 코드: 대상 항목을 0 ~ 9 까지 분할하고 필요한 만큼 반복
- 표의 숫자 코드: 항목의 성질의 물리적인 성질을 그대로 코드에 적용시키는 방법
- 합성 코드: 하나의 코드로 수행하기 어려운 경우 2개 이상의 코드를 조합하여 적용시키는 방법

### 결합도

종류

- 자료 결합도: 모듈간 인터페이스가 자료 요소로만 구성되느 제일 바람직
- 스탬프 결합도: 모듈간 인터페이스로 배열 등 자료구조가 전달될 때의 결합도
- 제어 결합도: 다른 모듈 내부의 흐름 제어를 위해서 통신하거나 제어 요소를 전달할 때의 결합도
- 외부 결합도: 어떤 모듈에서 선언한 데이터를 외부 다른 모듈에서 참조할 때의 결합도
- 공통 결합도: 공유되는 공통 데이터의 영역을 여러 모듈이 사용할 때의 결합도
- 내용 결합도: 다른 모듈의 내부기능이나 내부자료를 직접 참조하거나 수정할 때의 결합도

### 응집도

핵심

- 기능은 순차적으로 통신해야 절차적으로 시간이 논리적이되고 우연이 낮음

종류

- 기능적 응집도: 단일 기능의 요소로 하나의 모듈을 구성하는 경우
- 순차적 응집도: 출력 데이터를 그 다음 입력 데이터로 활용하는 경우
- 통신적 응집도: 동일 입출력을 사용, 서로 다른 기능을 수행하는 요소들이 모였을 경우 응집도
- 절차절 응집도: 모듈 내부 구성 요소들이 특정 기능을 순차적으로 수행할 경우의 응집도
- 시간적 응집도: 특정 시간에 처리되는 기능을 모아 하나의 모듈로 작성할 경우의 응집도
- 논리적 응집도: 유사한 성격을 갖는 처리 요소들로 하나의 모듈이 형성되는 경우의 응집도
- 우연적 응집도: 모듈 내부의 각 구성 요소들이 서로 관련없는 요소로만 구성된 경우의 응집도

## 소프트웨어 개발

### EAI (Enterprise Application Integration)

유형

- Point-to-Point
  - 가장 기본적인 1 대 1연결 방식
- Hub & Spoke
  - 단인 접점인 허브 시스템을 통해서 데이터를 전송하는 중앙 집중형 방식
- Message Bus
  - 애플리케이션 사이에 미들웨어를 두어 처리하는 방식
- Hybrid
  - Hub & Spoke와 Message Bus의 혼합 방식이다.
  - 그룹 내에서는 Hub & Spoke를, 그룹 간에는 Message Bus방식을사용한다.

순서

- 상향식

- 알파 검사: 개발자의 장소에서 사용자가 시험하고 개발자는 뒤에서 결과를 지켜보는 검사

- 테스트 케이스

## 데이터 베이스 구축

### 트랙잭션

- 데이터베이스의 상태를 변화시키는 논리적 연산의 집합

특징

- 원자성 (Atomicity)
  - 모두 반영되거나 아니면 전혀 반영되지 아니어야 한다.
- 일관성(Consistency)
  - 트랜잭션이 성공하면 언제나 일관적인 DB 상태로 변환
- 독립성(isolation)
  - 둘 이상의 트랜잭션이 동시에 병행 실행되고 있을 때 새로운 트랜잭션이 낄수 없다.
- 영구성(Diribility)

### 반정규화

- 정규화된 데이터 모델이 시스템의 성능향상, 개발 과정의 편의성, 운영의 단순화를 목적으로 수행되는 의도적인 정규화 원칙 위배 행위를 의미
- 데이터 모델을 중복, 통합, 분리 하는 과정

### E-R Diagram

- 사각형: 개체
- 마름모: 관계
- 타원: 속성
- 밑줄: 기본키
- 선: 연결, 링크

### 키

- 슈퍼키: 한 릴레이션 내에 속성들의 집합으로 구성된 키를 말함 (유일성)
- 후보키: 한 릴레이션 내에 모든 튜플들을 유일하게 식별할 수 있는 하나 또는 몇개의 애트리뷰트 집합 (최소 슈퍼키: 유일성 + 최소성)
- 기본키: 후보키 중에 선택한 키
- 대체키: 후보키 중에서 기본키를 제외한 속성들

### DCL(데이터 제어어)

- 데이터 관리를 목적으로 보안, 회복, 사용자 권한 등을 정의한다.

종류

- GRANT(권한부여) ON TO
- REVOKE (권한취소)
- COMMIT(트랜잭션 완료)
- ROLLBACK (트랜잭션 취소)

### 로킹

개요

- 주요 데이터들의 엑세스를 상호 베타적으로 하는 것

단위

- 병행제어에서 한꺼번에 로킹할 수 있는 데이터 단위
  - 데이터베이스, 파일, 레코드, 필드 드으이 로킹 단위가 될 수 있음
- 로킹 단위가 그면 로킹 수가 작아 관리하기가 쉽지만 병행성 수준이 낮아짐
- 로킹 단위가 작으면 로킹 수가 많아 관리하기 복잡해지지만 병행성 수준이 높아짐 (오버헤드가 커지지만 공유도가 증가함

### TCP 흐름제어기법

- Slow Start: 패킷이 문제 없이 도착하면 혼잡 윈도우 크기를 패킷마다 1을 증가시켜 한 주기가 지나면 혼잡 윈도우
- Slding Window: 한 번에 여러 패킷을 전송할 수 있어 전송 효율이 좋은 기법
  - 긍정 수신 응답(ACK)이 전달된 경우 윈도우의 크기는 증가, 아니면 감소
- Stop And Wait: 프레임이 손실된 경우 손실된 프레임 1개를 전송하고 수신자의 응답을 기다리는 방식으로 한번에 프레임 1개만 전송할 수 있는 기보ㅓㅂ

- DAS: 하드디스크와 같은 데이터 저장장치를 호스트버스 어댑터에 직접 연결하는 방식

### 정보의 접근통제

- NAC: 망 접근 제어
  - 내부망에 접속하는 단말의 보안성을 강화
- MAC: 어떤 주체가 특정 객체에 접근 시 양자의 보안레벨(등급)에 기초하여 낮은 수준의 주체가 높은 수준의 객체의정보에 접근하는 것을 제한하는 방법

### 다중화 기술

- Frequency Division Multiplexing(FDM, 주파수 분할 다중화): 통신 회선의주파수를 여러 개로 분할하는 다중화 방식
- Code Division Multiplexing(CDM, 코드 분할 다중화): 코드를 이용하여 신호를 다중화하는 방식
- TIme Division Multiplexing(TDM, 시간 분할 다중화): 시간 폭(Time slot)으로 나누어 여러 대의 단말 장치가 동시에 사용할 수 있도록 한 방식

### 컴포넌트 기반 방법론(CBD)

- 소프트웨어를 구성하는 컴포넌트를 조립해서 하나의 새로운 응용 프로그램을 작성하는 방법론
- 소스 코드 형태의 재사용과  바이너리 파일 형태의 재사용
- 클래스 컴포넌트
- 인터페이스를 통ㄹ한 접근
- 개발과 접근

### N - S Chart

- 순서도와는 달리 논리기술에 중점을 두고 상자도형을 이용한 도형식 설계도구로 순차(연속), 선택, 반복, 다중 선택 제어 구조를 표현하는 도구
